# Agent 轨迹评估说明

Agent 的结果不只看最终回答，还要看执行轨迹。

## 要评估什么

| 维度 | 示例 |
| --- | --- |
| 工具选择 | 查实时信息时是否调用搜索工具 |
| 工具顺序 | 是否先检索文档，再总结 |
| 参数正确性 | 调用天气工具时 city 是否正确 |
| 循环次数 | 是否陷入重复调用 |
| 最终答案 | 是否引用了检索证据 |

## 最小评估用例

```python
case = {
    "input": "查询北京天气并总结一句话",
    "expected_tools": ["get_weather"],
    "expected_keywords": ["北京", "天气"],
}
```

## 实践建议

1. Tool 单元测试先保证工具本身可靠。
2. Agent 集成测试用 fake model 固定路径。
3. 真实模型测试只覆盖关键路径，不追求每次完全相同。
4. 用 LangSmith trace 查看每次工具调用参数和中间消息。
5. 对失败样本建立数据集，持续回归。
